# 记忆系统各文档结构模板 (3/3) — 同步/归档/方法论/状态

## memory/[项目名]-sync/（项目文档镜像）

### 目录结构

```
[项目名]-sync/
├── .last-sync-ts              ← epoch 秒，上次同步时间
├── [项目]-architecture.md     ← 架构文档副本
├── [项目]-rules.md            ← 项目规则副本
├── [项目]-status.md           ← 状态副本
├── [项目]-decisions.md        ← 决策记录副本
└── [项目]-lessons.md          ← 项目踩坑副本
```

### 特点

- **单向同步**：项目 `docs/` → 这里
- **只读快照**，不反向修改
- `.last-sync-ts` 用于健康检查判断是否过期

### .last-sync-ts 示例

```
1772064066
```

（Unix epoch 时间戳，秒）

## memory/heartbeat-state.json

```json
{
  "lastChecks": {
    "email": null,
    "calendar": null,
    "weather": null,
    "projectSync": 1772064066
  },
  "note": "timestamps in epoch seconds"
}
```

### 说明

- `null` = 从未检查
- 数字 = 上次检查的 epoch 秒

### 更新示例

```bash
# 更新 projectSync 时间戳
NOW=$(date +%s)
jq ".lastChecks.projectSync = $NOW" heartbeat-state.json > tmp.json
mv tmp.json heartbeat-state.json
```

### methodology-subagent-development.md

子代理开发调度方法：

```markdown
# 子代理开发调度方法论

## 何时使用子代理

- 代码生成（>100 行）
- PR 审查
- 大规模重构
- 独立模块开发

## 代理选择

| 任务类型 | 推荐代理 | 理由 |
|----------|----------|------|
| Python/JS 开发 | Claude Code | 官方，稳定 |
| 探索性编程 | Codex | 快速原型 |
| 多语言项目 | Codex | 支持广 |

## 调度流程

1. 创建临时工作目录
2. 准备上下文文档（PRD + TASK）
3. spawn 子代理
4. 监控进度
5. 验证结果
6. 合并到主项目

## 上下文传递

- PRD → 子代理任务描述
- RULES.md → 子代理系统提示
- 项目文档 → 子代理 workspace
```
