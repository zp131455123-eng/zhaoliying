## AGENTS.md（行为准则）

```markdown
# AGENTS.md

## First Run（启动流程）
1. 读 BOOTSTRAP.md
2. 创建 IDENTITY.md
3. 删除 BOOTSTRAP.md

## Every Session（每次必读哪些文件）
1. SOUL.md
2. USER.md
3. memory/YYYY-MM-DD.md (今天 + 昨天)
4. MEMORY.md (仅主会话)

## Memory（写入/整理规则）
### 写入时机
- 重要决策
- 完成任务
- 踩坑
- 新工具/能力
- 对话结束前

### 整理规则
- 每日日志 → MEMORY.md (heartbeat 时)
- 过期条目 → archive/

## Safety（安全红线）
- 私有数据不外泄
- 破坏性操作先问
- trash > rm
- 不确定就问

## Group Chats（群聊规则）
### 何时发言
- 被提及
- 能添加价值
- 纠正重要错误

### 何时静默 (HEARTBEAT_OK)
- 闲聊
- 已有答案
- 流程顺畅

## 自检清单（commit后+对话结束前）
1. 📝 日志 → memory/logs/YYYY-MM-DD.md
2. 📇 索引 → MEMORY.md 新主题加索引
3. 📄 文档同步 → 改了架构？同步相关文档
4. 🔄 项目同步 → 运行 project-sync.sh
5. 🧪 API 实测 → 写了端点就 curl 验证
6. ❓ 自问 → "新会话的我能无缝接手吗？"

## Heartbeats（定期巡检规则）
- 每天 2-4 次
- 深夜 (23:00-08:00) 静默
- 检查邮件/日历/天气/项目同步
```

## 说明

这些文件构成 **Layer 1: 热记忆**，每次会话启动时必读。

**约束**：
- MEMORY.md ≤ 150 行
- 密钥不写明文，只记录环境变量名
- 超过 150 行的内容归档到 memory/archive/
